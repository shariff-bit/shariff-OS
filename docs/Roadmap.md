# Roadmap

## Development Strategy

Build Android-first.

The first objective is not to build every feature. The first objective is to make Shariff OS useful enough to run Shariff's day.

## Sprint 0: Foundation

Goal: create the product and architecture foundation.

Deliverables:

- product context
- MVP PRD
- architecture direction
- database model
- AI COO behavior
- UI/UX direction
- roadmap
- decision log
- README

Status: in progress

## Sprint 1: Flutter App Shell

Goal: create the Android-first Flutter app foundation.

Deliverables:

- Flutter project inside `frontend/`
- dark Material 3 theme
- app navigation
- feature folder structure
- Mission Control placeholder
- core mission model

## Sprint 2: Mission Control and Daily Planning

Goal: make the app useful in the morning.

Deliverables:

- Mission Control screen
- daily plan model
- top five priority flow
- mission assignment
- local draft state

## Sprint 3: Evening Review and Life Score

Goal: close the daily loop.

Deliverables:

- evening review screen
- priority completion tracking
- mood entry
- default Life Score calculation
- Life Score explanation

## Sprint 4: Supabase Backend

Goal: persist real data.

Deliverables:

- Supabase schema
- auth setup
- repository layer
- task persistence
- journal persistence
- review persistence

## Sprint 5: AI COO v1

Goal: add useful AI intelligence.

Deliverables:

- morning planning prompt
- evening review prompt
- journal summary prompt
- AI response storage
- backend/edge function pattern for AI calls

## Sprint 6: Android Usage and Notifications

Goal: connect phone behavior to execution.

Deliverables:

- Android usage permission flow
- usage stats reader
- target app usage summary
- morning notification
- evening notification

## Later

- weekly board meeting
- job tracker
- finance tracker
- news digest
- mascot state
- WHOOP integration
- Health Connect
- desktop and web

## Current Priority

Finish Sprint 0, then create the Flutter Android app shell.
