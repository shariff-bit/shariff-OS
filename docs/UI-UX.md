# UI and UX

## Direction

Shariff OS should feel like a premium dark mission control system.

It should be calm, fast, focused, and serious. The design should help Shariff execute, not decorate data.

## First Screen

The first screen is Mission Control.

It should answer:

> What should Shariff do next?

Primary areas:

- current day state
- next best action
- top priorities
- Life Score
- recovery/sleep signal
- mission progress
- AI COO advice

## Visual Style

- dark theme
- premium contrast
- restrained color
- no bright playful palette
- minimal clutter
- smooth motion
- compact information density
- Material 3 components adapted for a mission-control feel

## Color Direction

Use a dark neutral base with controlled accents.

Suggested accents:

- electric cyan for active systems
- muted green for recovery/progress
- amber for warnings
- red only for serious misses or risk

Avoid making the whole app one color.

## Navigation

MVP navigation should be simple:

- Mission Control
- Plan
- Review
- Journal
- Missions
- More

Do not bury the next action behind multiple taps.

## Components

Expected components:

- mission chips
- priority cards
- Life Score ring or meter
- review prompts
- mood selector
- journal editor
- AI COO panel
- usage summary
- bottom navigation

Cards should be used for real content blocks, not decorative nesting.

## Mascot

The penguin mascot is part of the COO experience.

It should appear when it adds meaning:

- celebrating completion
- warning about drift
- giving a short recommendation
- making review feel less dry

It should not clutter Mission Control.

## Copywriting

Use clear, direct product language.

Good:

- Next best action
- Mission risk
- Finish this before checking Instagram
- Board review

Avoid:

- generic productivity quotes
- long explanations
- motivational filler

## Android UX

The Android app should feel native and responsive.

Important:

- fast app launch
- clear notification actions
- readable dark mode
- one-hand-friendly planning and review
- no spam notifications

## Accessibility

- readable text contrast
- tappable controls large enough for phone use
- avoid relying only on color
- keep journal and review forms calm and easy to complete
